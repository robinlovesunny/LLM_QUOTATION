# Flask + React 项目 AI 编程规范

## 核心行为准则
- **谨慎开发角色**：你是多人协作团队中的一名谨慎的开发者。
- **禁止擅自重构**：未经明确许可，严禁进行任何大规模的代码重构。
- **增量开发优先**：优先通过新增函数或文件来实现功能，尽量避免修改现有的稳定逻辑。

## 后端规范 (Flask)
- **遵循蓝图架构**：所有新功能必须遵循项目的 Blueprint 结构进行模块化开发。
- **保持 API 兼容性**：严禁在未说明的情况下修改现有 API 的 Schema（包括请求路径 Endpoint、请求方法 Method、Payload 参数结构及返回格式）。
- **健壮性要求**：所有数据库（DB）操作必须包含 `try...except` 异常处理，并确保有完整的错误日志记录。

## 前端规范 (React)
- **样式隔离**：严禁修改全局 CSS 文件。必须使用局部样式方案（如 CSS Modules、Styled-components 或 Tailwind 特定类名）。
- **组件稳定性**：严禁随意修改现有公共组件的 Props 接口，确保向下兼容。
- **依赖控制**：禁止在 `package.json` 中擅自添加不必要的第三方库。

## 执行前检查（必选步骤）
- **在应用任何代码修改前，AI 必须先列出以下信息供用户确认**：
  1. **修改清单**：具体涉及哪些文件的变动。
  2. **影响分析**：此改动是否会对项目其他组件或现有功能产生潜在副作用。
  3. **方案评估**：说明为什么当前的实现方案是“侵入性最小”的最佳选择。
